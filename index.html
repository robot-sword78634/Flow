Flow

Friendly Language of Wonder

What is Flow?

Flow is a small, friendly, educational programming language inspired by classic 1970s and early 1980s computing. It is designed to make programming approachable, readable, and fun.

Flow leans heavily into the simplicity of early programming languages such as BASIC, but utilizes a model based on visual flowchart logic rather than line numbers. The language is equipped with a robust set of mathematical functions that should suit most users' needs and is capable of handling simple string variables. The idea behind the creation of Flow was to create an interpreted language that was low barrier for individuals who were new to programming and would inspire them to push the language's limitations to build new and interesting things. In short, to create a "friendly" language that would inspire wonder and interest in learning programming concepts. Essentially the same experience I had when first sitting in front of a computer in 1984 and realizing that I now had a set of tools that I could use to create whatever I could imagine.

Flow encourages learning through experimentation, creativity, and play rather than complexity or performance.

Philosophy

  • Simple syntax that is easy to read and understand
  • Intuituve design for people new to programming
  • Designed for games, simulations, and experiments
  • Intentional limitations that encourage creativity
  • Learning by doing

What Can You Make With Flow?

  • Text adventure games
  • Classic computer simulations (like Lunar Lander)
  • Math tutors and learning tools
  • Interactive demos and showcases
  • Terminal-based games and experiments

Downloading & Compiling Flow

Clone the Flow repository from GitHub: 

git clone https://github.com/robot-sword78634/flow.git
cd flow

Compile the interpreter (example using g++): 

g++ -std=c++11 flow.cpp -o flow

(Adjust filenames and compiler options as needed for your system.)

Hello World in Flow

Create a simple text file called hello.flow:

print "Hello, world!"

Run it using the Flow interpreter:

./flow hello.flow

Congratulations — you’ve written your first Flow program!

The History of Flow

    The development of Flow grew out of both a modest success and a spectacular failure. I had some limited success creating a simple Lisp interpreter, followed by a frustrating attempt to expand my skills by building a modern version of BASIC. That experience led me to realize that a more approachable project would be to design a language that captured the best ideas from early programming languages, combined with a small selection of concepts that make modern languages more accessible.

    As I began mapping out Flow, it became clear that a language which reads more like pseudocode than traditional source code could serve as a useful teaching tool for introducing people to computer programming. Much of Flow's syntax consists of simplified, stripped-down versions of constructs found in modern languages, with the goal that programs should "flow" naturally, much like the logic paths in a flowchart

    Flow is written in C++ and implemented as an interpreted language. This makes it portable, easy to experiment with, and approachable for learners, without requiring them to get too close to the underlying hardware or system details.

    Some future ideas for implementation include:

  • Expanding ASCII output abilities
  • Adding a wait function
  • Implementing arrays/lists
  • Building the ability to create user defined functions
